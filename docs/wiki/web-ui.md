# 网页界面与订阅流程

本文档说明当前 Web UI 的页面结构、交互流程，以及本次新增的分页、已订阅视图和手动 RSS 功能。

## 页面概览

当前网页端的核心入口是收藏看板，围绕两个主要组件展开：

- `CollectionBoard.vue`：展示 Bangumi 收藏列表、分页、已订阅状态和详情抽屉入口
- `BangumiDetailDrawer.vue`：展示番剧详情、Mikan 搜索结果、字幕组资源与手动 RSS 输入区块

后端通过 `src/server/main.ts` 将 `frontend/dist` 作为静态资源对外提供，因此生产或常规运行时，前后端可以通过同一个 `7810` 端口访问。

## 收藏看板

### 顶部 Tab

收藏看板目前包含 4 个分类：

| Tab | 数据含义 | 说明 |
|------|----------|------|
| 想看 | Bangumi 收藏类型 `1` | 计划观看 |
| 在看 | Bangumi 收藏类型 `3` | 当前正在观看 |
| 看过 | Bangumi 收藏类型 `2` | 已看完 |
| 已订阅 | 本地 RSS 源与收藏交集 | 用于查看已经建立订阅关系的番剧 |

### 未配置状态

如果未配置 Bangumi Token，看板不会直接报错，而是展示一个空态卡片，引导用户进入设置页完成配置。

### 分页行为

本次新增了仿 Firefly 风格的分页控件：

- 每页固定加载 30 条收藏
- 使用页码按钮、上一页、下一页和省略号
- 当前页高亮显示
- 首尾页自动禁用对应方向按钮
- 当总数不足一页时，分页区域自动隐藏
- 切换 Tab 时会自动重置到第 1 页

分页数据来自 `GET /api/bangumi/collections?type=&offset=&limit=`。

### 已订阅视图

“已订阅” Tab 的作用不是展示所有 RSS 源，而是展示“本地已经订阅，并且能匹配到 Bangumi 收藏条目”的番剧卡片。

当前交互流程如下：

1. 前端读取本地 RSS 源列表
2. 从 RSS 源中提取 `bangumiSubjectId`
3. 再读取 Bangumi 收藏列表
4. 以前两者的交集过滤出最终卡片

该视图默认隐藏分页导航，强调“结果列表”而不是“原始收藏分页”。

## 番剧详情抽屉

点击任意番剧卡片后，会打开右侧详情抽屉。

### 左侧信息区

左侧展示的是基础元信息：

- 海报
- 中文名 / 原名
- 评分
- 放送日期或年份
- 简介
- 跳转到 Bangumi 条目页的链接

### 右侧交互区

右侧包含两条订阅路径：

1. 通过 Mikan 搜索并选择字幕组订阅
2. 直接手动输入 RSS 地址订阅

---

## Mikan 搜索与字幕组订阅

抽屉打开后，会自动用番剧名称发起一次 Mikan 搜索。

### 搜索结果区

搜索结果会以卡片形式展示，点击某个 Mikan 条目后，前端会请求：

- `GET /api/mikan/search?q=`
- `GET /api/mikan/bangumi-detail/:id`

### 字幕组资源区

选中某个 Mikan 条目后，会展示字幕组列表和最近资源：

- 每个字幕组展示“发现 N 个资源”
- 点击字幕组后展开资源表格
- 表格列包含：番组名、大小、更新时间

为了兼容旧格式和新格式的 Mikan 返回数据，前端会统一按 `episodes[]` 结构渲染，避免出现“只有标题没有资源表”的情况。

### 高级配置

在字幕组订阅区可以配置：

- 包含正则
- 排除正则
- 集数偏移

确认订阅后，前端会调用 `POST /api/subscriptions` 创建高层订阅，并由后端同时落地 RSS 源与过滤规则。

## 手动 RSS 订阅

这是本次新增的核心功能之一。

### 交互入口

在详情抽屉底部新增了“手动输入 RSS”折叠区块。

### 输入项

手动 RSS 目前支持：

- RSS URL（必填）
- 包含正则（可选）
- 排除正则（可选）

### 校验与反馈

- RSS URL 必须以 `http://` 或 `https://` 开头
- URL 不合法时按钮会被禁用，并显示红色提示文案

### 保存与更新

手动 RSS 走统一的 `POST /api/subscriptions`：

- 首次保存时，会创建 `Manual RSS - {bangumiId}` 命名的 RSS 源
- 再次打开同一番剧抽屉时，前端会自动读取已有手动订阅并回填表单
- 如果已存在手动 RSS，按钮文案会从“保存订阅”变为“更新订阅”
- 更新时，后端会复用已有 source，并同步调整 include/exclude 规则

### 保存成功后的 UI 行为

保存成功后会立即执行以下动作：

1. 关闭详情抽屉
2. 触发收藏看板刷新
3. 更新卡片上的订阅状态
4. 让“已订阅” Tab 可见对应番剧

## 当前推荐使用流程

### 场景 1：通过 Mikan 订阅

1. 在收藏看板中找到番剧并打开抽屉
2. 选择正确的 Mikan 条目
3. 选择字幕组
4. 视需要填写正则和集数偏移
5. 点击“确认订阅”

### 场景 2：手动补录 RSS

1. 打开番剧详情抽屉
2. 展开“手动输入 RSS”
3. 粘贴可用 RSS URL
4. 视需要填写 include / exclude 正则
5. 点击“保存订阅”或“更新订阅”

## 与本次改动直接相关的 UI 结果

这次改动之后，网页端新增或修正了以下能力：

- 收藏列表支持分页浏览
- 新增“已订阅”分类视图
- 详情抽屉支持展示字幕组资源数量与大小信息
- 手动 RSS 支持保存、回填和更新
- Mikan 详情接口统一返回 `episodes[]`，减少前端兼容分支
